يبدأ CLI الخاص بـ Blackbox AI افتراضيا واجهة [TUI](/docs/tui) عند تشغيله بدون أي معاملات.

```bash
blackbox ai
```

كما يدعم أيضا أوامر موثقة في هذه الصفحة، ما يتيح لك التفاعل مع Blackbox AI برمجيا.

```bash
blackbox ai run "Explain how closures work in JavaScript"
```

---

### tui

بدء واجهة المستخدم terminal (TUI) الخاصة بـ Blackbox AI.

```bash
blackbox ai [project]
```

#### الخيارات

| الخيار       | المختصر | الوصف                                                             |
| ------------ | ------- | ----------------------------------------------------------------- |
| `--continue` | `-c`    | متابعة الجلسة الأخيرة                                             |
| `--session`  | `-s`    | معرّف الجلسة للمتابعة                                             |
| `--fork`     |         | تفريع الجلسة عند المتابعة (يستخدم مع `--continue` أو `--session`) |
| `--prompt`   |         | الموجّه المراد استخدامه                                           |
| `--model`    | `-m`    | النموذج المراد استخدامه بصيغة provider/model                      |
| `--agent`    |         | الوكيل المراد استخدامه                                            |
| `--port`     |         | المنفذ الذي يتم الاستماع عليه                                     |
| `--hostname` |         | اسم المضيف الذي يتم الاستماع عليه                                 |

## الأوامر

يتضمن CLI الخاص بـ Blackbox AI أيضا الأوامر التالية.

### agent

إدارة وكلاء Blackbox AI.

```bash
blackbox ai agent [command]
```

### attach

إرفاق terminal بخادم الواجهة الخلفية لـ Blackbox AI قيد التشغيل بالفعل، والذي تم تشغيله عبر الأمرين `serve` أو `web`.

```bash
blackbox ai attach [url]
```

يتيح ذلك استخدام واجهة TUI مع واجهة خلفية لـ Blackbox AI تعمل عن بعد. على سبيل المثال:

```bash
# ابدأ خادم الواجهة الخلفية للوصول عبر الويب/الجوال
blackbox ai web --port 4096 --hostname 0.0.0.0

# في محطة طرفية (terminal) أخرى، اربط TUI بالواجهة الخلفية قيد التشغيل
blackbox ai attach http://10.20.30.40:4096
```

#### الرايات

| الراية      | المختصر | الوصف                               |
| ----------- | ------- | ----------------------------------- |
| `--dir`     |         | دليل العمل الذي ستبدأ منه واجهة TUI |
| `--session` | `-s`    | معرّف الجلسة للمتابعة               |

### auth

أمر لإدارة بيانات الاعتماد وتسجيل الدخول لمزودي الخدمة.

```bash
blackbox ai auth [command]
```

---

#### login

يعتمد Blackbox AI على قائمة المزودين في [Models.dev](https://models.dev)، لذا يمكنك استخدام `blackbox ai auth login` لتهيئة مفاتيح API لأي مزود ترغب باستخدامه. يتم حفظ ذلك في `~/.local/share/blackbox ai/auth.json`.

```bash
blackbox ai auth login
```

عند تشغيل Blackbox AI يقوم بتحميل المزودين من ملف بيانات الاعتماد، وكذلك أي مفاتيح معرّفة في متغيرات البيئة لديك أو في ملف `.env` ضمن مشروعك.

---

#### list

يسرد جميع المزودين الذين تم توثيقهم كما هم محفوظون في ملف بيانات الاعتماد.

```bash
blackbox ai auth list
```

أو النسخة المختصرة.

```bash
blackbox ai auth ls
```

---

#### logout

يسجلك خارج مزود عبر حذفه من ملف بيانات الاعتماد.

```bash
blackbox ai auth logout
```


#### list

```bash
blackbox ai mcp ls
```

#### auth

إجراء المصادقة مع خادم MCP يدعم OAuth.

```bash
blackbox ai mcp auth [name]
```

إذا لم تُحدِّد اسم خادم، فسيُطلب منك الاختيار من الخوادم المتاحة الداعمة لـ OAuth.

يمكنك أيضا عرض الخوادم الداعمة لـ OAuth وحالة المصادقة الخاصة بها.

```bash
blackbox ai mcp auth list
```

أو استخدم النسخة المختصرة.

```bash
blackbox ai mcp auth ls
```

#### logout


### models

عرض جميع النماذج المتاحة من المزودين المُهيأين.

```bash
blackbox ai models [provider]
```

يعرض هذا الأمر جميع النماذج المتاحة عبر المزودين المُهيأين لديك بصيغة `provider/model`.

هذا مفيد لتحديد اسم النموذج الدقيق الذي ستستخدمه في [تهيئتك](/docs/config/).

يمكنك اختياريا تمرير معرّف مزود لتصفية النماذج حسب ذلك المزود.

```bash
blackbox ai models anthropic
```

#### الرايات

| الراية      | الوصف                                                         |
| ----------- | ------------------------------------------------------------- |
| `--refresh` | تحديث ذاكرة التخزين المؤقت للنماذج من models.dev              |
| `--verbose` | استخدام مخرجات أكثر تفصيلا للنماذج (تشمل بيانات مثل التكاليف) |

استخدم الراية `--refresh` لتحديث قائمة النماذج المخزنة مؤقتا. يفيد ذلك عند إضافة نماذج جديدة إلى مزود وتريد رؤيتها في Blackbox AI.

```bash
blackbox ai models --refresh
```

### run

تشغيل blackbox ai في وضع غير تفاعلي عبر تمرير موجه مباشرة.

```bash
blackbox ai run [message..]
```

هذا مفيد للبرمجة النصية والأتمتة، أو عندما تريد إجابة سريعة دون تشغيل واجهة TUI كاملة. على سبيل المثال.

```bash "blackbox ai run"
blackbox ai run Explain the use of context in Go
```

يمكنك أيضا الإرفاق بمثيل `blackbox ai serve` قيد التشغيل لتجنّب زمن الإقلاع البارد لخوادم MCP في كل تشغيل:

```bash
# ابدأ خادمًا بلا واجهة في محطة طرفية واحدة
blackbox ai serve

# في محطة طرفية أخرى، شغّل الأوامر التي ترتبط به
blackbox ai run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### الرايات

| الراية       | المختصر | الوصف                                                             |
| ------------ | ------- | ----------------------------------------------------------------- |
| `--command`  |         | الأمر المراد تشغيله؛ استخدم الرسالة كوسائط                        |
| `--continue` | `-c`    | متابعة الجلسة الأخيرة                                             |
| `--session`  | `-s`    | معرّف الجلسة للمتابعة                                             |
| `--fork`     |         | تفريع الجلسة عند المتابعة (يستخدم مع `--continue` أو `--session`) |
| `--share`    |         | مشاركة الجلسة                                                     |
| `--model`    | `-m`    | النموذج المراد استخدامه بصيغة provider/model                      |
| `--agent`    |         | الوكيل المراد استخدامه                                            |
| `--file`     | `-f`    | ملف/ملفات لإرفاقها بالرسالة                                       |
| `--format`   |         | التنسيق: default (منسق) أو json (أحداث JSON خام)                  |
| `--title`    |         | عنوان للجلسة (يستخدم موجهًا مقتطعًا إن لم تُحدَّد قيمة)           |
| `--attach`   |         | الإرفاق بخادم blackbox ai قيد التشغيل (مثل http://localhost:4096)    |
| `--port`     |         | منفذ الخادم المحلي (الافتراضي منفذ عشوائي)                        |

### serve

بدء خادم Blackbox AI بدون واجهة للوصول عبر API. راجع [توثيق الخادم](/docs/server) لواجهة HTTP الكاملة.

```bash
blackbox ai serve
```

يشغّل هذا خادم HTTP يوفّر وصولا عبر API لوظائف blackbox ai دون واجهة TUI. اضبط `BLACKBOX_AI_SERVER_PASSWORD` لتفعيل مصادقة HTTP الأساسية (اسم المستخدم الافتراضي `blackbox ai`).

#### الرايات

| الراية       | الوصف                                     |
| ------------ | ----------------------------------------- |
| `--port`     | المنفذ الذي يتم الاستماع عليه             |
| `--hostname` | اسم المضيف الذي يتم الاستماع عليه         |
| `--mdns`     | تفعيل اكتشاف mDNS                         |
| `--cors`     | أصول/منشأات إضافية للمتصفح للسماح بـ CORS |

#### list

```bash
blackbox ai session list
```

### stats

عرض إحصاءات استخدام الرموز والتكلفة لجلسات Blackbox AI لديك.

```bash
blackbox ai stats
```

#### الرايات

| الراية      | الوصف                                                                     |
| ----------- | ------------------------------------------------------------------------- |
| `--days`    | عرض الإحصاءات لآخر N يومًا (الافتراضي: كل الوقت)                          |
| `--tools`   | عدد الأدوات المطلوب عرضها (الافتراضي: الكل)                               |
| `--models`  | عرض تفصيل استخدام النماذج (مخفي افتراضيا). مرّر رقمًا لعرض أعلى N         |
| `--project` | التصفية حسب المشروع (الافتراضي: كل المشاريع، سلسلة فارغة: المشروع الحالي) |

```bash
blackbox ai export [sessionID]
```

### import

استيراد بيانات الجلسة من ملف JSON أو رابط مشاركة Blackbox AI.

```bash
blackbox ai import <file>
```

```bash
blackbox ai import session.json
blackbox ai import https://opncd.ai/s/abc123
```

| `--mdns`     | تفعيل اكتشاف mDNS                         |

### acp

بدء خادم ACP (Agent Client Protocol).

```bash
blackbox ai acp
```

يشغّل هذا الأمر خادم ACP يتواصل عبر stdin/stdout باستخدام nd-JSON.

#### الرايات

| الراية       | الوصف                             |
| ------------ | --------------------------------- |
| `--cwd`      | دليل العمل                        |
| `--port`     | المنفذ الذي يتم الاستماع عليه     |
| `--hostname` | اسم المضيف الذي يتم الاستماع عليه |

### uninstall

إلغاء تثبيت Blackbox AI وإزالة جميع الملفات ذات الصلة.

```bash
blackbox ai uninstall
```

#### الرايات

| الراية          | المختصر | الوصف                               |
| --------------- | ------- | ----------------------------------- |
| `--keep-config` | `-c`    | الإبقاء على ملفات التهيئة           |
| `--keep-data`   | `-d`    | الإبقاء على بيانات الجلسات واللقطات |
| `--dry-run`     |         | عرض ما سيتم حذفه دون تنفيذ الحذف    |
| `--force`       | `-f`    | تخطي مطالبات التأكيد                |

### upgrade

تحديث blackbox ai إلى أحدث إصدار أو إلى إصدار محدد.

```bash
blackbox ai upgrade [target]
```

للترقية إلى أحدث إصدار.

```bash
blackbox ai upgrade
```

للترقية إلى إصدار محدد.

```bash
blackbox ai upgrade v0.1.48
```

#### الرايات

| الراية     | المختصر | الوصف                                                       |
| ---------- | ------- | ----------------------------------------------------------- |
| `--method` | `-m`    | طريقة التثبيت المستخدمة: curl أو npm أو pnpm أو bun أو brew |

## متغيرات البيئة

| `BLACKBOX_AI_DISABLE_AUTOUPDATE`         | boolean | تعطيل التحقق التلقائي من التحديثات                        |
| `BLACKBOX_AI_DISABLE_PRUNE`              | boolean | تعطيل تنقية البيانات القديمة                              |
| `BLACKBOX_AI_DISABLE_TERMINAL_TITLE`     | boolean | تعطيل تحديث عنوان terminal تلقائيا                        |
| `BLACKBOX_AI_PERMISSION`                 | string  | تهيئة أذونات JSON مُضمّنة                                 |
| `BLACKBOX_AI_DISABLE_DEFAULT_PLUGINS`    | boolean | تعطيل الإضافات الافتراضية                                 |
| `BLACKBOX_AI_DISABLE_LSP_DOWNLOAD`       | boolean | تعطيل تنزيل خوادم LSP تلقائيا                             |
| `BLACKBOX_AI_ENABLE_EXPERIMENTAL_MODELS` | boolean | تفعيل النماذج التجريبية                                   |
| `BLACKBOX_AI_DISABLE_AUTOCOMPACT`        | boolean | تعطيل ضغط السياق تلقائيا                                  |
| `BLACKBOX_AI_DISABLE_CLAUDE_CODE`        | boolean | تعطيل القراءة من `.claude` (الموجه + المهارات)            |
| `BLACKBOX_AI_DISABLE_CLAUDE_CODE_PROMPT` | boolean | تعطيل قراءة `~/.claude/CLAUDE.md`                         |
| `BLACKBOX_AI_DISABLE_CLAUDE_CODE_SKILLS` | boolean | تعطيل تحميل `.claude/skills`                              |
| `BLACKBOX_AI_DISABLE_MODELS_FETCH`       | boolean | تعطيل جلب النماذج من مصادر بعيدة                          |

| `BLACKBOX_AI_DISABLE_FILETIME_CHECK`     | boolean | تعطيل التحقق من وقت الملف لتحسين الأداء                   |
| `BLACKBOX_AI_CLIENT`                     | string  | معرّف العميل (الافتراضي `cli`)                            |
| `BLACKBOX_AI_ENABLE_EXA`                 | boolean | تفعيل أدوات بحث الويب من Exa                              |
| `BLACKBOX_AI_SERVER_PASSWORD`            | string  | تفعيل المصادقة الأساسية لخادمي `serve`/`web`              |
| `BLACKBOX_AI_SERVER_USERNAME`            | string  | تجاوز اسم مستخدم المصادقة الأساسية (الافتراضي `blackbox ai`) |
| `BLACKBOX_AI_MODELS_URL`                 | string  | رابط مخصص لجلب تهيئة النماذج                              |
